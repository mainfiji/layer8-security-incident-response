# Обеспечение информационной безопасности в кадровом агентстве

## О проекте

Кадровое агентство Layer 8 занимается поиском IT-специалистов и хранит большой объём данных о кандидатах и заказчиках. В рамках проектной работы я:

1. **Устранил массовые блокировки пользователей** сайта компании, вызванные избыточными ограничениями WAF.
2. **Организовал доступ внешнему разработчику** к стенду разработки через LDAP и VPN.
3. **Проанализировал архитектуру VPN-шлюза** и сформулировал рекомендации по усилению безопасности.
4. **Настроил блокировку IP-адресов** из открытых списков угроз (IOC) через nftables.
5. **Подготовил отчёт об уязвимостях** (47 CVE) и план их устранения.
6. **Расследовал инцидент с утечкой данных** из CRM-системы.
7. **Осуществил реагирование** на подозрительные активности на хостах (изоляция, анализ логов).
8. **Разработал план перехода** на новый WAF (BunkerWeb) и подготовил документацию.

## Выполненные задачи

### Часть 1. Реагирование на инциденты

| Инцидент | Действие | Результат |
|----------|----------|-----------|
| Массовые блокировки пользователей | Настройка rate limiting с исключениями для пользовательского сегмента | Легитимные пользователи перестали блокироваться |
| Подозрительная активность на pc_vkrutikov | Изоляция через отключение VPN-пира | Хост изолирован, дальнейшее расследование передано администратору |
| Подозрительная активность на srv_waf и srv_ldap | Полная изоляция критичных серверов, сбор доказательств | Серверы выведены из эксплуатации, восстановлены из бэкапов |

### Часть 2. Управление уязвимостями

- Обнаружено **47 уникальных CVE** на 7 хостах.
- Топ-5 критических уязвимостей:
  - CVE-2024-6387 (regreSSHion) — 4 хоста
  - CVE-2024-3094 (XZ backdoor) — 2 хоста
  - CVE-2024-4762 (use-after-free в ядре) — 3 хоста
  - CVE-2023-4911 (Looney Tunables) — 4 хоста
- Разработан план устранения с указанием ответственных и сроков.

### Часть 3. Переход на новый WAF

| Этап | Результат |
|------|-----------|
| Планирование | Дорожная карта на 10 дней, распределение ролей |
| Анализ | Сравнение BunkerWeb, SafeLine, ModSecurity — выбран BunkerWeb |
| Проектирование | Размещение в DMZ, модули: WAF, антибот, rate limiting, GeoIP |
| Установка | Docker-контейнер BunkerWeb с настройками через переменные окружения |
| Тестирование | GoTestWAF: 94% SQLi, 90% XSS, 97% LFI |
| Документация | Инструкция по мониторингу для администраторов и SOC |

### Часть 4. VPN Security

| Вопрос | Ответ |
|--------|-------|
| Может ли пользователь менять конфигурацию? | Да, через параметр AllowedIPs в WireGuard |
| Возможно ли разграничение прав доступа? | Нет, все клиенты в одной подсети wg0 |
| Изолированы ли пользователи друг от друга? | Нет, есть риск горизонтального перемещения |
| Рекомендации | Внедрение SSL VPN с политиками, разделение на пулы, правила nftables для изоляции |
