# Рекомендации по улучшению безопасности VPN

## 1. Внедрение SSL VPN с поддержкой ролевых политик

Заменить WireGuard на решение, поддерживающее аутентификацию по LDAP и назначение прав после подключения.

**Варианты:**
- **OpenVPN Access Server** (коммерческая, но есть бесплатная версия на 2 подключения)
- **WireGuard + FreeRADIUS** (сообщественный подход)
- **StrongSwan** (IPsec с аутентификацией по LDAP)

**Преимущества:**
- Возможность назначать разные IP-пулы для разных групп пользователей
- Применение политик на уровне межсетевого экрана на основе IP-адресов из этих пулов

---

## 2. Разделение VPN-пулов по ролям

Создать несколько виртуальных интерфейсов на VPN-шлюзе:

| Интерфейс | Подсеть | Роль | Доступные сегменты |
|-----------|---------|------|---------------------|
| wg_dev | 10.10.20.0/24 | Разработчики | 192.168.6.0/24 (только стенд разработки) |
| wg_admin | 10.10.10.0/24 | Администраторы | Все сегменты (192.168.0.0/16) |
| wg_users | 10.10.30.0/24 | Обычные сотрудники | 192.168.2.0/24 (только файловый сервер и Wiki) |

На межсетевом экране `fw` добавить правила, разрешающие доступ только из соответствующих подсетей к целевым сегментам.

**Пример правил nftables:**

```bash
# Разрешить разработчикам доступ только к сегменту разработки
nft add rule inet filter forward ip saddr 10.10.20.0/24 ip daddr 192.168.6.0/24 accept
nft add rule inet filter forward ip saddr 10.10.20.0/24 drop

# Разрешить администраторам доступ ко всем сегментам
nft add rule inet filter forward ip saddr 10.10.10.0/24 accept

# Разрешить обычным сотрудникам доступ только к файловому серверу
nft add rule inet filter forward ip saddr 10.10.30.0/24 ip daddr 192.168.1.60 accept
nft add rule inet filter forward ip saddr 10.10.30.0/24 drop
```

## 3. Изоляция клиентов друг от друга

На VPN-шлюзе добавить правило, запрещающее прямой трафик между клиентами одной подсети:

```bash
nft add rule inet filter forward ip saddr 10.10.10.0/24 ip daddr 10.10.10.0/24 drop
```

Это предотвратит горизонтальное перемещение.

---

## 4. Внедрение Zero Trust Network Access (ZTNA)

Рассмотреть использование прокси-агентов на конечных точках (например, Cloudflare Zero Trust, Zscaler, или OpenZiti). В этом случае доступ к ресурсам предоставляется на основе аутентификации пользователя, а не IP-адреса.

**Преимущества:**
- Микросервисная сегментация
- Доступ только к конкретным приложениям, а не ко всей сети
- Журналирование всех действий

---

## 5. Усиление аутентификации

- Внедрить двухфакторную аутентификацию (TOTP, Push) для всех VPN-подключений.
- Настроить политику паролей (сложность, срок действия) через LDAP.
- Запретить использование простых паролей.

---

## 6. Централизованный аудит и мониторинг

- Настроить отправку логов WireGuard (или нового VPN-решения) в Wazuh.
- Создать правила корреляции для выявления подозрительных подключений (например, вход в нерабочее время, многократные неудачные попытки).
- Включить оповещения при изменении конфигурации клиента (изменение AllowedIPs).

## Заключение
