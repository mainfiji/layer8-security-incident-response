# Анализ требований к новому WAF

## Исходные данные

Текущий WAF (связка Nginx + ModSecurity) имеет следующие недостатки:
- Отсутствие графического интерфейса
- Сложность анализа логов (только текстовые файлы)
- Высокая нагрузка на CPU из-за большого количества правил (регулярных выражений)
- Много ложных срабатываний
- Сложность интеграции с современными SIEM-системами

## Ключевые требования (сформулированы командой ИБ)

### 1. Графический интерфейс
- Визуализация атак в реальном времени
- Автоматическая группировка однотипных инцидентов (агрегация логов)
- Упрощение анализа логов для оперативного реагирования

### 2. Продвинутый анализ трафика
- Семантический анализ (понимание контекста кода), а не просто поиск по регулярным выражениям
- Снижение нагрузки на CPU
- Минимизация ложных срабатываний

### 3. Высокая производительность
- Минимальная задержка при обработке запросов
- Высокая пропускная способность при DDoS-атаках уровня L7

### 4. Модель лицензирования
- Бесплатное или условно-бесплатное решение (с пробным периодом, с бесплатными базовыми функциями)
- Работа под ОС Linux

### 5. Защита от ботов
- Поведенческий анализ
- Блокировка по User-Agent, TLS fingerprint

### 6. Географическая фильтрация
- Блокировка по странам (GeoIP)

### 7. Контейнеризация
- Поддержка Docker для гибкого масштабирования

### 8. Интеграция с Wazuh
- Отправка логов в формате JSON через syslog
- Возможность создания декодеров и правил корреляции

---

## Приоритезация требований

| Приоритет | Требование |
|-----------|------------|
| Высокий | Графический интерфейс, семантический анализ, защита от ботов, интеграция с Wazuh |
| Средний | Производительность, работа в Docker |
| Низкий | GeoIP блокировка (уже есть в текущей архитектуре) |

---

## Дополнительные пожелания

- Простота установки и настройки (предпочтительно через Docker Compose)
- Хорошая документация (официальная и сообщество)
- Возможность кастомизации правил
- Наличие встроенных механизмов обновления сигнатур

---

## Вывод

Основные требования: GUI, семантический анализ, защита от ботов, интеграция с Wazuh, бесплатность. Все эти критерии должны быть учтены при выборе нового WAF (см. сравнительную таблицу в `vendor_comparison.md`).
