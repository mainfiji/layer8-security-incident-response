# Проектирование внедрения нового WAF (BunkerWeb)

## Размещение в сети

Новый WAF будет размещён в сегменте **DMZ** (192.168.4.0/24) на хосте `srv_waf_new` (IP 192.168.4.15). Он заменит старый WAF, работающий на том же сегменте.

**Топология:**
Интернет → Пограничный МСЭ (fw) → DNAT на порт 80/443 → srv_waf_new (BunkerWeb, 192.168.4.15) → Reverse Proxy → srv_web (192.168.1.10)

## Модули WAF и их назначение

| Модуль | Назначение |
|--------|------------|
| **Основной WAF (OWASP CRS)** | Анализ HTTP/HTTPS трафика, блокировка известных атак (SQLi, XSS, LFI, RCE) |
| **Защита от ботов** | Поведенческий анализ, блокировка подозрительных User-Agent, CAPTCHA при превышении порога |
| **Rate Limiting** | Ограничение частоты запросов по IP, защита от DDoS и brute-force |
| **GeoIP Filtering** | Блокировка или ограничение доступа из стран с высоким уровнем риска |
| **HTTP-аутентификация** | Дополнительная защита административных панелей (Basic Auth) |
| **Reverse Proxy** | Проксирование запросов к бэкенд-серверам (srv_web, другие веб-приложения) |

## Регулярное обслуживание WAF

| Периодичность | Действие |
|---------------|----------|
| Ежедневно | Мониторинг дашборда (атаки, ошибки, нагрузка) |
| Еженедельно | Обновление правил OWASP CRS (автоматически через встроенный механизм) |
| Ежемесячно | Анализ ложных срабатываний, корректировка исключений |
| Ежеквартально | Тестирование с GoTestWAF |
| Постоянно | Резервное копирование конфигурации (volumes) |

## Карта здоровья WAF (Health Check)

Мониторинг следующих метрик:
- **Доступность:** HTTP-запрос на страницу /health
- **Нагрузка:** RPS (запросов в секунду), latency
- **Блокировки:** количество заблокированных запросов, top атакующих IP
- **Бэкенды:** статус доступности srv_web
- **Логи:** ошибки в контейнере (через Wazuh)

## Интеграция с Wazuh

### Этапы интеграции:
1. **Настройка отправки логов из BunkerWeb в syslog:**
   - В веб-интерфейсе BunkerWeb: Settings → Logs → Enable syslog
   - Указать IP Wazuh-менеджера (192.168.3.10) и порт 514 (UDP)
   - Выбрать формат JSON

2. **Настройка приёма в Wazuh:**
   - В `/var/ossec/etc/ossec.conf` добавить секцию `<remote>` для syslog
   - Перезапустить Wazuh-менеджер

3. **Создание декодеров и правил:**
   - Декодеры для парсинга JSON-логов BunkerWeb (пример в `configs/wazuh_decoders_rules.xml`)
   - Правила корреляции: блокировка SQLi, XSS, ботов, high severity

4. **Визуализация:**
   - Добавить дашборд для BunkerWeb в Wazuh (можно через Kibana)

## План переключения на новый WAF

1. Установить BunkerWeb на новый хост (Docker-контейнер).
2. Настроить Reverse Proxy на srv_web.
3. В режиме "только логирование" (без блокировки) проверить работу в течение 24 часов.
4. Переключить DNAT на межсетевом экране на новый WAF.
5. Включить блокировку (режим защиты).
6. Провести тестирование GoTestWAF.
7. Настроить интеграцию с Wazuh.
8. Вывести старый WAF из эксплуатации (архивировать логи).
