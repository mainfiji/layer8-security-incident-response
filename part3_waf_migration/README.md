# Часть 3. Переход на новый WAF

## Обзор

В компании принято решение о замене старого WAF (связка Nginx + ModSecurity) на более современное решение. Основные причины:
- Отсутствие графического интерфейса
- Сложность анализа логов (только текстовые файлы)
- Высокая нагрузка на CPU из-за большого количества правил
- Много ложных срабатываний

## Анализ решений

| Критерий | BunkerWeb | SafeLine | ModSecurity (старый) |
|----------|-----------|----------|---------------------|
| Графический интерфейс (GUI) | Есть (веб-интерфейс для мониторинга) | Полноценный веб-интерфейс с дашбордами | Нет (только текстовые логи) |
| Визуализация и агрегация логов | Базовая визуализация, требуется SIEM | Развитая система агрегации | Требуется внешний SIEM |
| Анализ трафика | Комбинация сигнатур и поведенческий анализ | Семантический анализ (SAW) | Регулярные выражения (OWASP CRS) |
| Производительность | Средняя | Низкая задержка, высокая пропускная способность | Высокая нагрузка на CPU |
| Бесплатность / Open Source | Есть бесплатная версия, открытый код | Есть бесплатная версия (Community) | Бесплатный, open source |
| Защита от ботов | Базовые механизмы (CAPTCHA, rate limiting) | Встроенная защита (анализ поведения) | Отсутствует |
| GeoIP блокировка | Поддерживается | Поддерживается | Поддерживается |
| Работа в Docker | Нативная поддержка | Официальная поддержка | Требуется ручная настройка |
| Интеграция с Wazuh | Логи в syslog, есть плагины | Логи в syslog, формат JSON | Сложный парсинг |

### Выбор решения

Выбрано решение **BunkerWeb** по следующим причинам:
1. Полностью бесплатный и open source
2. Графический интерфейс для мониторинга
3. Семантический анализ (снижает нагрузку и ложные срабатывания)
4. Встроенная защита от ботов
5. Поддержка Docker и интеграции с Wazuh
6. Лучшая документация по сравнению с SafeLine

## Проектирование

### Размещение в сети
Новый WAF размещается в сегменте **DMZ** (192.168.4.0/24), аналогично старому WAF.

### Модули WAF
| Модуль | Назначение |
|--------|------------|
| Основной модуль WAF | Анализ HTTP/HTTPS трафика с OWASP CRS |
| Защита от ботов | Поведенческий анализ, блокировка подозрительных ботов |
| Rate Limiting | Ограничение частоты запросов (DDoS, brute-force) |
| GeoIP Filtering | Блокировка трафика из нежелательных стран |
| HTTP-аутентификация | Дополнительная защита административных панелей |

### Обслуживание WAF
- **Ежедневно:** мониторинг дашборда
- **Еженедельно:** обновление правил (автоматически через CRS)
- **Ежемесячно:** анализ ложных срабатываний
- **Ежеквартально:** тестирование GoTestWAF
- **Регулярно:** резервное копирование конфигурации

### Интеграция с Wazuh
- Настройка отправки логов BunkerWeb в формате JSON на Wazuh-менеджер через syslog
- Создание декодеров для парсинга событий
- Создание правил корреляции для алертов
