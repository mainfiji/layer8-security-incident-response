# Инструкция: Мониторинг событий безопасности WAF BunkerWeb

## 1. Введение

Данная инструкция предназначена для администраторов и аналитиков SOC, отвечающих за мониторинг и реагирование на инциденты безопасности, связанные с веб-приложениями, защищёнными с помощью WAF BunkerWeb.

## 2. Доступ к веб-интерфейсу

BunkerWeb предоставляет веб-интерфейс (UI) для мониторинга и управления. По умолчанию доступен по порту **9443** (HTTPS).
https://<IP_адрес_WAF>:9443

text

Учётные данные задаются при установке (переменные `WEB_UI_LOGIN` и `WEB_UI_PASSWORD`).

## 3. Основные разделы мониторинга

### 3.1 Dashboard (Панель управления)
- Общее количество запросов (всего, успешных, заблокированных)
- График нагрузки (запросы в секунду)
- Топ-5 атакующих IP
- Типы атак (круговая диаграмма)
- Статус бэкендов

### 3.2 Attacks (Атаки)
Детализированная информация о каждом заблокированном запросе:
- Время, IP-адрес атакующего, тип атаки, URI
- Фильтрация по дате, IP, типу атаки, правилу
- Просмотр полных заголовков и тела запроса
- Экспорт в CSV или JSON

### 3.3 Requests (Запросы)
Все запросы, прошедшие через WAF (разрешённые и заблокированные):
- Поиск конкретных запросов
- Анализ поведения ботов
- Проверка ложных срабатываний

### 3.4 Logs (Журналы)
Просмотр логов WAF в реальном времени (хвост логов).
Настройка уровня детализации (info, warn, error).

## 4. Ключевые метрики

| Метрика | Что показывает | Норма | Действие при отклонении |
|---------|---------------|-------|------------------------|
| Количество заблокированных запросов | Уровень атакующей активности | Зависит от профиля | Анализировать источник, тип атак |
| Топ атакующих IP | Наиболее активные источники атак | Должно быть немного адресов | Заблокировать на файрволе |
| Типы атак | Преобладающие векторы атак | — | Проверить уязвимость приложения |
| RPS (Requests per second) | Нагрузка на WAF | Соответствовать мощности | Масштабирование или настройка лимитов |
| Latency | Задержка, вносимая WAF | Не более 50-100 мс | Проверить производительность |

## 5. Настройка уведомлений

### Email-уведомления
Settings → Notifications → Enable email notifications
- Параметры SMTP-сервера
- Адреса получателей
- Выбор событий для уведомлений

### Webhook
Settings → Notifications → Enable webhook
- URL эндпоинта
- Секретный токен
- Формат payload (JSON)

## 6. Интеграция с Wazuh

### 6.1 Настройка отправки логов в syslog
Settings → Logs → Send logs to syslog
- IP Wazuh-менеджера и порт (UDP 514)
- Формат: JSON

### 6.2 Настройка приёма в Wazuh
В `/var/ossec/etc/ossec.conf`:
```xml
<remote>
    <connection>syslog</connection>
    <port>514</port>
    <protocol>udp</protocol>
    <allowed-ips>192.168.0.0/16</allowed-ips>
</remote>
